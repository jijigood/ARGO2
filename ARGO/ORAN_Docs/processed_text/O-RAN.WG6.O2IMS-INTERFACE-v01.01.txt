2.1.1.1.3 ResourceType 238 
The ResourceType service object represents a resource type in an O-Cloud. 239 
Resource types provide the categorization of O-Cloud Nodes as described in the O-RAN Cloud Architecture and 240 
Deployment Scenarios [7] specification. The values for these types are used to build the inventory and enable an 241 
authorized consumer to track resource changes over time. The O-Cloud provides a mechanism to discover the resource 242 
types with respective operations on resource types. 243

2.1.1.1.4 ResourcePool 247 
The ResourcePool service object respresents a resource pool instance in an O -Cloud. 248 
The O-Cloud is a distributed cloud with resources placed in different locations. Those resources are organized into 249 
resource pools. There can be more than one resource pool for a given O-Cloud at the same physical location. However, 250 
each resource pool requires a different identity since the SMO will direct a deployment based on homing policies to a 251 
specific location. This is different than typical cloud deployments as location is key to meeting latency requirements, 252 
sometime to elements outside of the O-Cloud. 253 
NOTE: The identity of a resource pool cannot be tied to a specific resource within the pool as that resource may 254 
be replaced as part of the O-Cloud lifecycle. 255 
The information representing the ResourcePool shall follow the provisions indicated in table 2.1.1.1.4 -1. 256

2.1.1.1.7 Subscription 279

when resources within the inventory change. 281 
Subscriptions are required for a notification to be generated. Information about the change are provided to the consumer 282
