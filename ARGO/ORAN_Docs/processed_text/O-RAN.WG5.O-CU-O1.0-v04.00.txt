This O1 interface facilitates the initialization, configuration and management of the O-CU-CP and O-CU-UP. Please refer 40 
to O-RAN OAM Architecture [7] for more details on the architecture. 41

O1 interface for O-CU-CP and O-CU-UP is used for supporting the management features including “start up” installation, 43 
PNF software management,  provisioning management, performance management, fault management and file 44 
management towards the O-CU.  45

and UE trace file, Performance Measurement result file, etc. The following file management use cases are supported for 4 
the O-CU-CP and O-CU-UP: 5 
• List available files 6 
• Transfer  file 7 
• Download file 8 
• File ready notification 9

diagram can be found in the O-RAN O1 interface specification [8], Chapter 2.5. 11

8.4 File Management Operation: transferFile 13 
The Transfer File Service provides the capability for the SMO to transfer files from or to the O-CU-CP and O-CU-UP.  14 
SMO/OAM may perform this action as a result of a FileReady notification from the O -CU-CP and O-CU-UP informing 15 
SMO that a file is now available to transfer. In any case, the file transfer is performed using a secure file transfer 16 
protocol (SFTP or FTPeS) from or to the location provided by O-CU-CP and O-CU-UP. The SMO triggers file transfer 17 
operation to O-CU-CP and O-CU-UP. Detailed procedure can be found in Chapter 2.5.3 of O1 Interface Specification 18 
[8]. The transferFile operation for O-CU-CP and O-CU-UP is performed independently. 19 
8.5 File Management Operation: downloadFile 20 
The Download File Service provides the capability for the SMO to request the O-CU-CP and O-CU-UP to download 21 
the file(s) when the SMO has a file that needs to be downloaded. In any case, the file download is performed using a 22 
secure file transfer protocol (SFTP or FTPeS) from the location provided by the SMO. The O-CU-CP and O-CU-UP 23 
replies to the SMO with the result of the download procedure. Detailed procedure can be found in Chapter 2.5.4 of O1 24 
Interface Specification [8]. The downloadFile operation for O-CU-CP and O-CU-UP is performed independently. 25
