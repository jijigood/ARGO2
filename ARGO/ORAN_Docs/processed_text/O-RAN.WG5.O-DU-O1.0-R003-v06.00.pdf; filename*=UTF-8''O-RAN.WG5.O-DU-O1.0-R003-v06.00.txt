9.3.1 Cell and carrier activation 6
O-DU sync-state and O-RU sync-state shall be LOCKED (or HOLDOVER) before O-DU is allowed to send gNB-DU 7
Configuration Update to CU, because it contains Served NR Cells To Add/Modify/Delete IE indicating cells which O-8

When O-DU receives GNB-CU CONFIGURATION UPDATE including Cells to be Activated List Item IE from CU, 10
O-DU shall check that O-DU sync-state and O-RU sync-state are LOCKED (or HOLDOVER) before O-DU configures 11
to O-RU the value of the parameter “active” at tx-array-carrier element / rx-array-carrier element to “ACTIVE” for the 12
carriers serving the cell(s) requested to be activated.  13
If O-DU sync-state is not LOCKED (and not HOLDOVER), then O-DU shall not activate any cells from Cells to be 14
Activated List Item IE. O-DU shall report back to CU Cells failed to be activated list IE.  15
If O-RU(s) sync-state is not LOCKED (and not HOLDOVER), then O-DU shall not activate carriers related to Cells to 16
be Activated List Item IE, to those O-RU(s) which sync-state is not LOCKED (and not HOLDOVER). O-DU shall 17
report back to CU Cells failed to be activated list IE for cells that O-DU did not activate carrier(s) to O-RU(s). 18

gNB-DU Configuration update Cells Status IE all O-DU cells as inactive. 21
When O-DU receives notification from O-RU that O-RU sync-state transit to FREERUN, O-DU shall disable all 22
carriers in that O-RU and report to CU in gNB-DU Configuration update Cells Status IE all affected cells as inactive. 23

Description The number of inbound on time messages with a correct ecpriSeqId (no

Condition The number of inbound on time messages with a correct ecpriSeqId (no
sequence number error) which are dropped by the terminating entity due
to the message containing one or more eCPRI/1914.3 or ORAN protocol
errors.
Protocol errors are defined as when eCPRI/1914.3/ORAN defined fields
contain invalid values or indicate unsupported capabilities.  Some
examples of this are:
1. PcId or section Id number which has not been configured.
2. Unexpected use of C bit,
3. Unconfigured or Unsupported udCompHdr setting.
4. Unsupported section extension.
5. Wrong ecpriVersion and/or payloadVersion information
6. ecpriMessage field does not contain 0, 2, or 5.
