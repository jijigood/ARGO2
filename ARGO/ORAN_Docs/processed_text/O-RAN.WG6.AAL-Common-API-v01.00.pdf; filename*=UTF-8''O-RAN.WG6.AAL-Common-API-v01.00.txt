Chapter 2 Overview 1
2.1 General 2
The AAL General Aspects and Principles is described in [3] including a high-level architecture of the AAL and 3
definition of the AAL profiles. The AAL interface has two distinct parts, the first part corresponds to a set of common 4
interfaces (‘AALI-C’). These are used to address all the profile independent aspects between application(s) and 5
underlying AALI Implementation(s) including management and orchestration within a O-Cloud platform. AAL Profile 6
specific interface (AALI-P) are a set of operations that are specific to each defined AAL profile.  The AALI-P are 7
addressed in FEC[8] and HIGH Phy specifications [9]. This document details the list of AALI Common (AALI-C) 8
operations and events, description with input/outputs, information models, state transition diagrams and requirements to 9
implement an AALI.  10
Within the AALI-C interface there are two distinct buckets: 11
 12
• AAL-C-Mgmt: Common adminstrative operations/actions/events toward O-Cloud IMS. The HW Accelerator 13
Manager terminates the AALI-C-MGMT interface. 14
• AAL-C-App: Common operations/actions/events toward RAN Application 15
 16
 17
2.2 Document Structure 18
This present document is structured as follows:  19
Chapter 2 presents the overview and main purpose of this specification.  20
Chapter 3 presents a list of requirements and object model.  21
Chapter 4 presents the initialization and flow, operations and events associated with AALI-C-Mgmt.  22
Chapter 5 presents the initialization and flow, operations and events associated with AALI-C-App.   23
Chapter 6 Appendix provides some informational state transition charts and examples of implementation.  24

3.4 Object model and scenarios 2
The AALI-C-Mgmt interface is the Management interface. It is used by the IMS for adminstrative 3
operations/actions/events. The HW Accelerator Manager terminates the AALI-C-MGMT interface.   4
Operations and actions/events via AALI-C-Mgmt are applicable on the following entities: 5
• HW Accelerator – Physical Managed Resource 6
• AAL Logical Processing Unit (AAL-LPU) – Logical Managed Resource 7
• AAL Profile Instance – Logical Managed Resource 8
The AALI-C-App interface is the application facing interacing. It is used by the application to discover, configure, and 9
manage the logical AAL resources assigned to the application.   10
Note: A Network Function or application may choose to use one or more AAL profiles as part of its implementation.  11
The AALI-C-App services has operations on the following abstract objects that are provided by the AAL 12
Implementation: 13
• AAL-LPU 14
• AAL-Profile-Instance 15
• AAL-Profile-Queue 16
 17
Operations and actions/events via AALI-C-App are applicable on the following entities: 18
• AAL Logical Processing Unit (AAL-LPU) – Logical Managed Resource 19
• AAL Profile Instance - Logical Managed Resource 20
• AAL Queue - Logical Managed Resource 21
 22
• HW accelerator: AAL Logical Processing Unit = 1: N2 (N2 >= 1) 23
• AAL Logical Processing Unit : AAL Profile Instance = 1: N3 (N3 >= 1) 24
• AAL Logical Processing Unit : AAL Queue = 1: N4 (N4 >= 1) 25

Chapter 5 AALI-C-APP  16
5.1 AALI-C-App Identification & Representation  17
Within a process address space each AAL-LPU can be abstracted using a generalize object handle. 18
Depending on HW design and implementation choice, an AAL-LPU implementation may want to accelerate multiple 19
AAL profiles, offer support for sharing HW Accelerator resources between multiple threads and or partition AAL-LPU 20
resources between multiple cells. For this reason, the second and third abstract constructs known as AAL-Profile-21
Instance and AAL Queue can optionally be used by the network function to  22
• distinguish between multiple supported AAL profiles per AAL-LPU 23
• prioritize access to AAL-LPU resources   24
• group operation requests 25
• partition AAL-LPU resources  26
• Allow parallel access through AALI for multiple threads / cores  27
As an abstract construct, an AAL Queue does not reflect a HW design specification or requirement but an AAL 28
interface specification.  29

                      O-RAN.WG6.AAL-Common API v01.00 TS

assigned to the NF, the NF shall initialize and configure each AAL-LPU that is assigned to it.  2
