Executive summary 6 
This document provides the description of the requirements and stage-2 information models to implement an 7 
Acceleration Abstraction Layer (AAL) for the High-PHY functionality hosted by an AAL-LPU that is associated with 8 
an O-RAN Distributed Unit (O-DU). Informative examples of stage-3 implementations are described in informational 9 
Annexes of this present document. 10 
Introduction 11 
This document specifies how to accelerate the High-PHY functionality of an O-DU where the High-PHY Acceleration 12 
Function (AF) is hosted on an AAL-LPU that is assigned to the AAL Application that is part of the O-DU. This 13 
document also specifies a set of AAL Profiles and procedures for utilizing the AALI Common Management (AALI-C-14 
Mgmt) and Application (AALI-C-App) interfaces to convey the information elements specified by the AAL Profiles. 15

This section contains information for each AAL Profile that is specified in this present document. An AAL Profile may 5 
provide APIs for a specific profile configuration that may be required in addition to the general AAL configuration 6 
APIs. AAL Profile specific configuration shall be done on a per AAL-LPU basis. Each AAL Profile shall define a list 7 
of capabilities that will be used to define the operations supported by the AAL -LPU.  8 
A normative stage-3 definition for this High-PHY Profile specification is FFS and could leverage aspects of various 9 
options including but not limited to SCF-222. 10
