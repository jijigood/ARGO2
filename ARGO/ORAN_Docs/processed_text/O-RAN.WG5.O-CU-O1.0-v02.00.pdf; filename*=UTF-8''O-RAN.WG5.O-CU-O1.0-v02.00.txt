Chapter 2 High Level Description 29
2.1 Top level functional description and terminology,  30
2.1.1 Architecture for O-RAN WG5 functional split 31
This O1 interface facilitates the initialization, configuration and management of the O-CU-CP and O-CU-UP. Please refer 32
to O-RAN OAM Architecture [7] for more details on the architecture. 33
2.1.2 O1 interface for O-CU architecture model 34
O1 interface for O-CU-CP and O-CU-UP is used for supporting the management features including “start up” installation, 35
software management, configuration management, performance management, fault management and file management 36
towards the O-CU.  37
2.1.3 Transport Network 38
Based on the transport topology, various modes of network connectivity are possible between O-CU-CP, O-CU-UP and 39
SMO, as defined in the O-RAN OAM Architecture document [7]  .  40
  41
