6 RAN Function Service Model Description 1 
6.1 RAN Function Overview 2 
E2 Service Model KPM (E2SM-KPM) supports O-CU-CP, O-CU-UP, and O-DU as part of NG-RAN connected to 3 
5GC or as part of E-UTRAN connected to EPC. 4 
The E2 Node shall host the RAN Function “KPM Monitor” which performs the following functionalities: 5 
- Exposure of available measurements from O-DU, O-CU-CP, and/or O-CU-UP via the RAN Function Definition IE. 6 
- Periodic reporting of measurements subscribed from Near-RT RIC. 7 
This E2SM specification also exposes a set of services described in clause 6.2. 8

- C ondition-based, UE-level E2 Node Measurement 14 
- Common Condition-based, UE-level E2 Node Measurement 15 
- E2 Node Measurements for multiple UEs 16 
These services may be initiated according to: 17 
- Periodical event. 18

7.4.2.4 REPORT Service RIC Indication Message IE contents 13 
This REPORT Service style uses the RIC Indication Message IE Format 1 (8.2.1.4.1). 14 
The REPORT Service RIC Indication Message IE carries a set of measurement data reported from an E2 Node . The 15 
reported data contains a set of measurement records, each collected at every granularity period during the reporting 16 
period. In case the E2 Node is not able to provide reliable data for a granularity period during the reporting period, it 17 
may include the optional Incomplete Flag IE, which indicates that the corresponding measurements record in the 18 
reported data is not reliable. 19 
The REPORT Service RIC Indication Message IE optionally carry subscription information, i.e. Measurement 20 
Information List IE that indicates the order of measured values for each measurement record in the reported data, or 21 
their granularity period. If not present, the original subscription information shall apply.  22
