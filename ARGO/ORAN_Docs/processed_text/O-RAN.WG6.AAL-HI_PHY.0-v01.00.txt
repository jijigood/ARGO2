Chapter 3 AALI Configuration and Management  1 
The AALI configuration and management APIs are the APIs that an application (O-DU) executes to configure and 2 
manage the AAL-LPU(s) that have been allocated to the application by the O-Cloud. The high-level Configuration and 3 
Management Principles are presented in [7].  4 
Chapter 4 AAL Profile Specifications 5 
 Profile Specifications Overview   6 
This section contains information for each AAL profile that is supported. A profile can provide several APIs for specific 7 
profile configuration that may be required in addition to the general AAL configuration APIs. Profile specific 8 
configuration shall be done on a per AAL-LPU basis. Each profile shall define a list of capabilities that will be used to 9 
define the operations supported by the AAL-LPU.  10 
 Naming  11 
Parameters and capabilities based on this stage-2 specification should follow the following naming guidelines: 12 
• Parameter and capability names shall start with a lowercase letter; 13 
• No spaces may be used in the names; 14 
• The names shall consist only of letters and digits; 15 
• The first letter of each non-initial word in a name shall be capitalized. 16 
 17 
 O-DU AAL_DOWNLINK_High-PHY Profile Specification 18 
This profile provides acceleration functionality for the following channels and signals:  19 
- PDSCH (including Data, DM-RS and PT-RS) 20 
- PDCCH (including Data and DM-RS) 21 
- CSI-RS 22 
- SSB (Including PSS, SSS and PBCH) 23 
This section presents parametrization for each of these channels and signals. 24 
 Profile Operation 25 
The AAL_DOWNLINK_High-PHY Profile interface shall work on a slot basis, where the slot, numerology and SFN 26 
are signaled for the API. 27 
The input data is specific to each downlink channel composing the profile. The output data is a set of C/U-Plane O-28 
RAN Fronthaul (OFH) packets [8]. 29 
The AAL-LPU interfaces directly with the OFH and is responsible for handling its exchange of information with OFH .  30
