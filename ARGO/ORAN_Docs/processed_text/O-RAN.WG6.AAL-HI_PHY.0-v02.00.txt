Chapter 3 AALI Configuration and Management  1 
The AALI configuration and management APIs are the APIs that an application (O-DU) executes to configure and 2 
manage the AAL-LPU(s) that have been allocated to the application by the O-Cloud. The high-level Configuration and 3 
Management Principles are presented in [7].  4 
Chapter 4 AAL Profile Specifications 5 
 Profile Specifications Overview   6 
This section contains information for each AAL profile that is supported. A profile can provide several APIs for specific 7 
profile configuration that may be required in addition to the general AAL configuration APIs. Profile specific 8 
configuration shall be done on a per AAL-LPU basis. Each profile shall define a list of capabilities that will be used to 9 
define the operations supported by the AAL-LPU.  10 
A normative stage 3 definition for this High-PHY Profile specification is FFS and could leverage aspects of various 11 
options including but not limited to SCF-222 [9]. 12 
An informative example of one possible mapping of the stage-2 HI-PHY Profile in this document is provided in Annex 13 
A (SCF-222 [9]). 14 
 Naming  15

 O-DU AAL_DOWNLINK_High-PHY Profile Specification 21 
This profile provides acceleration functionality for the following channels and signals: 22 
- PDSCH (including Data, DM-RS and PT-RS) 23 
- PDCCH (including Data and DM-RS) 24 
- CSI-RS 25 
- SSB (Including PSS, SSS and PBCH) 26 
This section presents parametrization for each of these channels and signals, and pertinent carrier configuration. 27 
 Profile Operation 28 
The AAL_DOWNLINK_High-PHY Profile interface shall work on a slot basis, where the slot, numerology and SFN 29 
are signaled for the API. 30 
The input data is specific to each downlink channel composing the profile. The output data is a set of C/U-Plane O-31 
RAN Fronthaul (OFH) packets [8]. 32 
The AAL-LPU interfaces directly with the OFH and is responsible for handling its exchange of information with OFH.  33 
As much as possible, parameter redundancy should be avoided. Where this is not possible, the AAL-LPU can assume 34 
that redundant parameters are consistent. 35

The AAL_DOWNLINK_High-PHY Profile capabilities shall be reported to the application.  37
